Flipkart Business Case

Detecting Cycles in Reviews

üìå Business Case

Flipkart wants to detect review cycles where sellers and buyers post fake reviews for mutual benefit using multiple fake accounts.

üß† Algorithms Used

  • Graph Traversal: Users are modeled as nodes, reviews as directed edges.
  • Cycle Detection (DFS / Union-Find): Detect tightly knit communities.
  • Connected Components: Identify suspicious clusters of accounts.

üß† Intuition

Fake review rings behave like a small network of users giving each other good ratings repeatedly, which can be flagged by detecting cycles.

üí• Impact

Reduces manipulation of product ratings and boosts trustworthiness of reviews for end customers.

üíª Sample Base Code (Cycle Detection - C++)

#include <iostream>
#include <vector>
using namespace std;

bool dfs(int node, vector<vector<int>>& graph, vector<bool>& visited, vector<bool>& recStack) {
    visited[node] = true;
    recStack[node] = true;

    for (int neighbor : graph[node]) {
        if (!visited[neighbor] && dfs(neighbor, graph, visited, recStack)) return true;
        else if (recStack[neighbor]) return true;
    }
    recStack[node] = false;
    return false;
}

bool hasCycle(vector<vector<int>>& graph, int n) {
    vector<bool> visited(n, false), recStack(n, false);
    for (int i = 0; i < n; i++)
        if (!visited[i] && dfs(i, graph, visited, recStack)) return true;
    return false;
}

int main() {
    int n = 4;
    vector<vector<int>> graph = {
        {1}, {2}, {0}, {2}
    };
    cout << (hasCycle(graph, n) ? "Cycle detected" : "No cycle") << endl;
    return 0;
}
