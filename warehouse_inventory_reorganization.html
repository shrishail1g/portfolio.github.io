Flipkart Business Case

Warehouse Inventory Reorganization

üìå Business Case

Flipkart wants to reorganize warehouse inventory such that the most frequently ordered products are stored in the most accessible locations. This helps reduce the time taken by staff to pick items during order fulfillment.

üß† Algorithms Used

  • Greedy Sorting: Sort products by order frequency or access time.
  • Heap / Priority Queue: Efficiently maintain top-k frequently accessed items.
  • Hash Maps: Count accesses for each SKU.

üß† Intuition

Like placing fast-moving grocery items near the store entrance, Flipkart reorders its warehouse layout based on real-time product popularity.

üí• Impact

Improves fulfillment efficiency, reduces labor costs, and supports faster delivery during peak sales events.

üíª Sample Base Code (Greedy Sorting - C++)

#include <iostream>
#include <vector>
#include <unordered_map>
#include <queue>
#include <algorithm>
using namespace std;

int main() {
    unordered_map<string, int> accessFrequency = {
        {"Phone", 120}, {"Charger", 300}, {"Laptop", 80},
        {"Shoes", 250}, {"T-shirt", 180}, {"Book", 220}
    };

    // Convert to vector for sorting
    vector<pair<string, int>> items(accessFrequency.begin(), accessFrequency.end());

    // Sort by frequency (descending)
    sort(items.begin(), items.end(), [](auto &a, auto &b) {
        return a.second > b.second;
    });

    cout << "Optimal Warehouse Placement Order:\n";
    for (auto &item : items) {
        cout << item.first << " (" << item.second << ")\n";
    }
    return 0;
}
